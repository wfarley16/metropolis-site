The idea

The hard part of any planning app — durable multi-step workflows, memory, criteria discovery, first-party research, approval-gated execution — is the same in every domain. What changes is the domain knowledge. So we built the planning engine once, and bring in a real domain expert to make each space genuinely great — which means we can stand up a planning app for almost any high-stakes decision.

The repeatable loop

Discover → Plan → Execute → Track

Backed by always-on research into the vault, and a shared design system so every city is beautiful by default.

1 · Discover

Elicit real criteria — self-discovery, not a search box — while researching the domain live.

2 · Plan

Crawl first-party sources, match with explanations, assemble a revisable plan.

3 · Execute

One click, with your approval: book it, order it, apply — reliably, and reversibly.

4 · Track

Receipts, confirmations, status — a living record that feeds the next decision.
